🚀 프로젝트 최종 정의: "Context-First Orchestration OS"
이 시스템은 단순한 AI 앱이 아니라, **"사용자의 의도(Intent)와 과거의 맥락(Context)을 결합하여 로컬/클라우드 환경에서 실무를 완벽하게 수행하고 검증하는 운영 체계"**입니다.

1. 존재의 목적 (Core Mission)
기존 AI: "말을 잘 듣는 인턴" (시키는 대로 하지만 맥락을 놓치고 실수를 함)

본 시스템: "나를 기억하는 수석 아키텍트" (과거의 결정, 정책, 스타일을 모두 기억하여 내가 없어도 내 방식대로 일을 처리하고 검증함)

2. 기억의 3층 구조 (Memory Hierarchy)
AI가 상상하지 않고 '근거'에 기반해 작동하게 만드는 뼈대입니다.

Semantic: "무엇을 했는가?" (유사 맥락 검색)

Structural: "어떻게 연결되어 있는가?" (모듈/인물 간의 관계망)

Policy: "무엇을 지켜야 하는가?" (절대 변하지 않는 시스템 규칙 및 불변성)

3. 구조적 정합성 수호 (Guardian Loop)
AI가 내놓은 결과물이 기존 PRD, 설계 결정(Decision), 정책과 충돌하는지 실행 전에 시스템이 먼저 검사합니다.

통제권: AI의 답변을 그대로 수용하는 것이 아니라, 시스템이 ALLOW/WARN/BLOCK 신호를 주어 인간의 최종 승인을 돕습니다.

4. 하이브리드 실행 환경 (Local-Cloud Link)
뇌 (Server): 기억과 정책을 관리하고 판단을 내립니다.

손발 (Local Agent): 실제 사용자의 PC 레포지토리에 접근하여 파일을 수정하고 터미널 명령을 실행합니다.

인터페이스 (Mobile/Web): 장소에 상관없이 "의도"만 던지면 PC가 알아서 실무를 수행하도록 지휘합니다.

5. 도메인 확장성 (Domain-Neutral Core)
Core Runtime은 아무런 편견이 없습니다. (무색무취)

여기에 [코딩 팩], [저술 팩], [PPT 팩] 등 도메인 번들을 끼우는 순간, 해당 분야의 전문가 비서로 변신합니다. 모든 도메인은 사용자의 공통된 '장기 기억'을 공유합니다.
