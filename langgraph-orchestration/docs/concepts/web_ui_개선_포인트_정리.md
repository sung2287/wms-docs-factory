# Web UI 개선 포인트 정리 (추천 우선순위)

목표:
> 일단 "채팅 중심"으로 충분히 쓸 수 있는 상태를 만드는 것.

기능 확장(워크플로우 고도화) 이전에, 사용성부터 안정화한다.

---

# 🔴 1순위 (지금 바로 체감되는 개선)

1. 채팅 히스토리 JSON → 말풍선 UI로 변경
- 현재: raw JSON 형태라 가독성 매우 낮음
- 개선: user / assistant 구분된 말풍선 렌더
- 기대효과: 디버그 화면 → 실제 채팅 화면으로 체감 전환

2. Provider 미설정 안내 배너
- 현재: LLM_PROVIDER 없으면 아무 반응 없는 것처럼 보임
- 개선: 상단에 빨간 안내 배너 표시
  - "LLM Provider가 설정되지 않았습니다"
  - 실행 예시 명령어 함께 표시
- 기대효과: 설정 문제로 막히는 상황 제거

---

# 🟡 2순위 (사용 피로도 감소)

3. 세션 UX 개선
- 현재 세션 입력 → Init 방식은 불편
- 개선:
  - 최근 세션 목록 표시
  - 현재 세션 강조 표시
  - 세션 삭제 버튼
- 기대효과: "채팅방" 개념이 명확해짐

4. Reset / Rerun 버튼 설명 추가
- 현재: 버튼 의미 직관적이지 않음
- 개선: 버튼 옆에 간단 설명 또는 툴팁 추가
- 기대효과: 실수 방지, 기능 이해도 상승

---

# 🟢 3순위 (앱 완성도 상승)

5. UI에서 Provider / Model 선택 가능하게
- 현재: 서버 실행 시 env로만 설정 가능
- 개선:
  - Provider 드롭다운
  - Model 선택 필드
- 기대효과: 터미널 의존도 제거

6. 상태 표시 개선
- 현재: Busy / Step 표시가 단순 텍스트
- 개선:
  - 진행 중 애니메이션
  - 현재 단계 시각 강조
- 기대효과: 실행 흐름 가시성 향상

---

# 방향 정리

지금 단계의 목표는:
> "워크플로우 실행기"가 아니라
> "멀티 프로바이더 ChatGPT 클론" 수준의 완성도 확보

UX 안정화 후에 워크플로우 시각화/자동화로 확장한다.
