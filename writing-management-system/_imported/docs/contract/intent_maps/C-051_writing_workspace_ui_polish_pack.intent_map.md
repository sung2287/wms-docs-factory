# C-051: Writing Workspace UI Polish Pack Intent Map
**Status: Draft**

## 1. WHY? (Motivation)
"우리는 도구의 시끄러움을 줄여, 글을 쓰는 몰입감을 방해하지 않고자 한다."

- **UI 소음 감소(Reduce UI Noise):** 문단마다 발생하는 스크롤바, 트리 탐색기의 중복된 이동 버튼(Up/Down), 상시 노출되는 좌/우 구분선 등은 사용자의 시선을 분산시킨다. 
- **일관된 미니멀리즘(Consistent Minimalism):** PRD-049에서 시작된 미니멀한 문서 작성 환경을 트리 탐색기와 워크스페이스 전체로 확장하여, 단일한 디자인 언어를 완성한다.
- **가독성 및 피로도 개선(Readability & Fatigue):** 다크 모드의 대비를 정교하게 조정하고 전역 테마 토큰을 도입하여 장시간 작업 시의 눈의 피로를 줄인다.
- **작성 환경의 유연성(Authoring Smoothness):** 문단이 내용에 따라 자동으로 늘어나는 환경은 물리적인 종이에 글을 쓰는 듯한 연속적인 느낌을 준다.

## 2. WHAT IS FIXED? (Constraints)
이번 설계를 통해 "반드시 지켜야 하는 시각적 및 상호작용 원칙"을 고정한다.

- **문단 자동 높이 조절(Auto-grow):** 개별 문단의 스크롤바 노출은 금지되며, 콘텐츠의 길이에 따라 UI가 유연하게 반응해야 한다.
- **트리 가독성 최우선:** 트리는 카드가 아닌 텍스트 기반의 리스트 형태로 표현되어야 하며, 드래그 핸들은 필요할 때만 노출한다.
- **상태 보존(Persistence):** 트리 노드의 접힘/펼침 상태는 브라우저 수준(`localStorage`)에서 기억되어야 하며, 이는 사용자의 워크플로우를 존중하는 기본 요소이다.
- **중앙 집중형 테마 관리:** 모든 색상 및 시각적 속성은 중앙 토큰 시스템을 경유해야 하며, 하드코딩된 값은 원칙적으로 금지한다.

## 3. WHAT IS NOT FIXED? (Flexibility)
- **코어 로직의 영구성:** 이번 설계는 UI 폴리싱에 국한되므로, 트리 엔진(PRD-048)이나 데이터 스냅샷 구조에는 영향을 주지 않는다. 추후 코어 엔진이 업그레이드되더라도 이 UI 원칙은 유지될 수 있도록 설계한다.
- **기능의 영구 제거 아님:** UI에서 Up/Down 버튼은 보이지 않지만, 내부 로직은 유지되므로 차후 접근성 모드나 다른 레이아웃에서 다시 활성화될 수 있는 가능성을 열어둔다.

## 4. DESIGN PHILOSOPHY
- **숨겨진 도구(Hidden Tools):** 도구는 상시 노출되는 대신, 사용자가 필요로 하는 시점(Hover, Focus)에만 세밀하게 나타나야 한다.
- **부드러운 상호작용(Soft Interaction):** 구분선 노출이나 스크롤바의 변화 등은 시각적 충격을 주지 않도록 부드럽고 미세하게 처리한다.
- **데이터와 뷰의 분리:** 트리 노드를 접는 행위는 사용자의 '보는 방식'에 대한 선택일 뿐, 실제 데이터의 위계나 순서를 건드리지 않아야 한다.
