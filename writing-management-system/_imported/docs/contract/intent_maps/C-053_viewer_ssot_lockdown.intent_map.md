# C-053: Viewer SSOT Lockdown Intent Map
**Status: Draft (Freeze Candidate)**

## 1. 의도 (Motivation)
"우리는 UI의 무분별한 증식을 막고, 변화의 통로를 단일화하여 시스템의 엔트로피를 제어하고자 한다."

- **유지보수 효율화:** 여러 버전의 뷰어가 공존하면서 발생하는 시각적 불일치와 중복된 버그 수정 비용을 근본적으로 제거한다.
- **예측 가능성 확보:** 새로운 기능을 추가할 때 기존 렌더러를 재사용하게 강제함으로써, 시스템 전체의 시각적/동작적 일관성을 보장한다.
- **전략적 통합:** PRD-052에서의 "UI 완성"을 넘어, 이를 아키텍처적으로 "보호"하는 단계를 확립한다.
- **Renderer Unity:** 본 정책은 Viewer의 수를 줄이는 것이 목적이 아니라, 렌더링 엔진(Renderer)의 단일성을 보장하는 것이 목적이다. UI Skin의 분리는 허용된다.

## 2. 전략적 의의 (Strategic Value)
- **분리와 정복:** "기능 개발(052)"과 "구조적 제어(053)"를 분리하여, 아키텍처 변경이 사용자 경험에 미치는 리스크를 최소화한다.
- **SSOT의 진화:** 데이터 레벨의 SSOT를 넘어, 표현 레벨(Renderer)의 SSOT를 달성함으로써 진정한 의미의 단일 진실 공급원을 완성한다.

## 3. 기대 효과
- 다중 뷰어 공존으로 인한 개발자 및 사용자의 인지 부하 감소.
- CI 가드레일을 통한 기술 부채 자동 방지.
- 생산용 빌드 아티팩트의 경량화 및 보안 강화 (레거시 코드 노출 차단).
- **Skin Benefits:** Skin 분리를 통해 디버그/제품 UI를 분리하면서도 렌더링 로직의 단일성을 유지할 수 있다.

## 4. 비의도 (Non-Intent)
- 새로운 UI 기능을 개발하거나 기존 인터랙션을 개선하려는 의도가 아니다.
- 핵심 엔진의 데이터 흐름이나 저장 정책을 변경하려는 시도가 아니다.
