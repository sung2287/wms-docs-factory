# C-049: Writing UI Minimal Document Mode Intent Map
**Status: Draft**

## 1. WHY? (Motivation)
"우리는 도구를 쓰고 싶지 않고, 글을 쓰고 싶어 한다."

- **문서 인지 모델:** 기존 블록 에디터는 '각 문단이 독립적인 블록'임을 강조하기 위해 카드 형태를 취한다. 하지만 이는 사용자가 글을 쓸 때 하나의 흐름으로 인식하는 것을 방해한다.
- **몰입감(Immersion):** 글을 쓰는 도중에 UI 요소가 상시 노출되어 있으면 시선이 분산된다. 
- **ChatGPT & Apple Aesthetics:** 사용자들이 선호하는 현대적 에디터는 '떠 있는 캔버스'와 '숨겨진 컨트롤'의 조합으로, 이를 WMS에도 적용하여 고급스러운 경험을 제공하려 한다.

## 2. WHAT IS FIXED? (Constraints)
이번 설계를 통해 "반드시 지켜야 하는 시각적 원칙"을 고정한다.

- **불필요한 UI 제거:** 문단의 번호, 상시 노출되는 버튼, 박스형 경계선 등은 원칙적으로 금지한다.
- **글로벌 바 원칙:** 사이드바와 테마 토글 등 시스템적 기능은 문서(Canvas) 영역 밖의 '글로벌 UI 바'에서만 제어한다. 이는 문서의 순수성을 보존한다.
- **저대비 원칙(Low Contrast):** 문단 편집 툴(Hover UI)이나 상태바는 배경과 대비가 낮아야 한다. 이는 텍스트 자체에 집중하게 만든다.

## 3. WHAT IS NOT FIXED? (Flexibility)
- **철학의 영구 고정 아님:** 이번 설계는 '미니멀 모드'에 집중하지만, 추후 다른 레이아웃 모드(예: 아웃라인 모드)가 추가될 수 있음을 전제한다.
- **코어 정책 영향 없음:** 데이터 저장, 스냅샷, 트리 구조 엔진 등의 로직은 049의 범위 밖이며, UI 레이어의 변경이 백엔드 계약을 깨지 않도록 설계되었다.

## 4. DESIGN PHILOSOPHY
- **마찰 없는 편집(Frictionless):** 편집 도구는 필요할 때(Hover)만 나타나고, 글을 읽을 때는 존재를 감춘다.
- **깊이감의 미학(Micro Depth):** 단순한 평면이 아니라 캔버스가 배경 위에 아주 살짝 떠 있는 느낌을 주어, "지금 내가 중요한 무언가를 작성하고 있다"는 심리적 만족감을 준다.
